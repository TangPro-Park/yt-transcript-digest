## 2. 위스키 DB 구축

인기 영상 #1~#11 다이제스트 파일 11개를 직접 읽어 테이스팅 분석 데이터베이스 작성.

### 추출 컬럼

| 컬럼 | 내용 |
|------|------|
| 위스키 이름 | 한글 정식명 |
| 화자 핵심 표현 | 호스트 직접 발언 |
| 추정 테이스트 노트 | 발언 기반 추정 |
| 위스키 배경 | 증류소·출시 정보 |
| 추정 점수 | /100 (발언 강도 기반 추정) |
| 타입 | 스카치/일본/한국 + 세부 캐스크 분류 |

### 점수 추정 방법론

호스트의 명시적 수치 없음 → 언어 강도에서 역산:
- 인생 위스키·면세 원픽 → 93~96
- "블루 뺨따구·차원이 다른" → 90~92
- "술술 들어간다·욕 안 먹는다" → 87~89
- "실망이다·나무 맛만" → 63~70

### 결과

- **총 17개 위스키** (11개 영상)
- 점수 범위: 63 (화요 XP) ~ 96 (히비키 21 화조풍월)
- 가성비 인사이트: 화요 53도 (84점, 면세 $60) > 화요 XP (63점, 면세 $100~120)
- 반전 픽: 킹조지5세 — 한때 저평가 → 재발견 후 91점

## 3. 파일 구조 정리

인기 영상 다이제스트 파일들이 채널 루트에 산재해 있던 것을 정리.

```
# 이동 전
output/주토피아/2021-10-30_글렌알라키_.md   ← 루트에 混在
output/주토피아/2024-02-22_신작_스프링뱅크_.md

# 이동 후
output/주토피아/인기/2021-10-30_글렌알라키_.md
output/주토피아/인기/reports/위스키_DB_전체.md
```

**작업 내용**:
- 인기 영상 md 파일 11개 → `인기/` 이동
- `인기/reports/` 서브디렉터리 신설
- DB 파일 3종 → `인기/reports/` 저장

## 4. 데모 HTML (whiskey_demo.html)

Cask & Codex 디자인 시스템 언어를 활용한 단독 실행 HTML 데모.

### 기술 스택
- Vanilla JS (CDN 불필요, 단독 실행 가능)
- Google Fonts (Noto Serif KR + Inter) — 인터넷 연결 시 로드
- 인라인 CSS — 외부 파일 의존 없음

### 디자인 토큰
| 역할 | 색상 |
|------|------|
| 배경 | `#f7f4ed` (bone paper) |
| 헤더/히어로 | `#2d5a3d` (bottle green) |
| 점수/악센트 | `#c4821a` (amber copper) |
| S티어(90+) | amber |
| A티어(80~89) | bottle green |
| B티어(70~79) | sage |
| C티어(~70) | muted red |

### 기능
- **리스트 뷰**: 17개 카드 그리드 (1/2/3열 반응형)
- **필터 탭**: 전체 / 스카치 / 일본 / 한국 / 티어별
- **상세 뷰** (카드 클릭 시):
  - 보틀 그린 히어로 밴드 (이름 + 점수)
  - 호스트 발언 quote 카드 (앰버 테두리)
  - 테이스팅 프로필 8축 바차트
  - 추정 노트 / 배경 / 최종 평가
  - 뒤로가기 버튼

### 실행 방법
- 로컬: 파일 더블클릭 → 브라우저 오픈 (JS 필수)
- 모바일: tiiny.host 등 무료 호스팅 → URL 접속 (카카오톡 내장 뷰어는 JS 비실행으로 내용 불표시)
