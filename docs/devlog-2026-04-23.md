## 4. 신규 기능 개발

### 4-1. 트랜스크립트 캐시 채널별 구조화

**변경 전**: `cache/transcripts/{video_id}.txt` (flat)  
**변경 후**: `cache/transcripts/{채널명}/{video_id}.txt` (채널별 폴더)

**이유**: 채널이 늘어날수록 flat 구조에서 파일 구분이 불가능해짐. 채널 단위 폴더가 output 미러링보다 안정적 (채널을 다른 카테고리로 재분류해도 캐시 경로 불변).

**변경 파일**: `main.py` `_build_entry()` 함수 (3줄)

```python
# 추가된 로직
channel_name = video.get('channel_name', 'unknown')
cache_dir = os.path.join(TRANSCRIPT_CACHE_DIR, sanitize_dirname(channel_name))
```

`transcript.py`는 이미 `cache_dir` 파라미터를 지원하므로 무변경.  
기존 flat 캐시 파일은 호환 유지 (채널폴더에 없으면 새로 받아옴).

---

### 4-2. Claude CLI subprocess 자동화 + 모델 선택

**기존**: `--llm claude`는 `pending.json`만 생성하고 Claude Code 세션이 수동으로 파일을 읽고 처리  
**변경**: `claude --model {model} --print` subprocess를 자동 호출, `--llm gemini`와 동일한 자동화 패턴

**신규 파일**: `src/claude_cli.py`

```python

    'opus':   'claude-opus-4-7',
}

def process_with_claude_cli(item, template_path, model_alias='haiku'):
    # 프롬프트 조립 후 claude --print subprocess 호출
    # stdin으로 프롬프트 전달 → 트랜스크립트 길이 제한 없음
```

**`main.py` 변경**:
- `_run_claude_processing(manifest, claude_model)` 추가
- `--claude-model` CLI 인자 추가 (default: `haiku`)
- `llm == 'claude'` 분기에서 자동 처리 실행

**CLI 사용법**:

```bash
# 기본 (Haiku — 빠름/저비용)
python main.py --channel "..." --start 2026-04-23 --mode medium --llm claude

# Sonnet
python main.py --url "..." --mode medium --llm claude --claude-model sonnet
```

**모델 선택 기본값을 Haiku로 설정한 이유**: 트랜스크립트 정리 작업은 지시 따르기(instruction-following) + 요약이 핵심으로, Sonnet 대비 비용 절감이 크고 품질 차이가 제한적임.
