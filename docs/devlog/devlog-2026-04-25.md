## 2. Haiku 에이전트 모드 오염 — 원인·수정

### 문제

`claude --print` subprocess로 Haiku를 호출하면 Claude Code가 CLAUDE.md를 읽고 에이전트 세션으로 동작. 출력 파일 본문에 아래 패턴이 포함된 채로 저장됨.

```
파일 쓰기 권한이 필요합니다 / 요청하시는지 명확히 해주시면 / What would you like me to do?
파일을 저장하겠습니다 / 기다리고 있습니다 / 분석 완료!
```

### 수정 (`src/claude_cli.py`)

```python
result = subprocess.run(
    [
        claude_exe,
        '--model', model_id,
        '--print',
        '--tools', '',                          # 에이전트 모드 차단
        '--system-prompt', '당신은 텍스트 생성기입니다. ...',  # 텍스트 생성기로 고정
    ],
    input=prompt,
    ...
)
```

- `--tools ""` : 도구 사용 비활성화
- `--system-prompt` : "마크다운 stdout만 출력"을 선언해 에이전트 루프 진입 차단

추가로 `_find_claude_exe()` 함수 신설 — PATH에 `claude` 없을 때 Windows glob으로 `claude.exe` 자동 탐색.

## 4. 채널 URL 헤더 링크

output 파일 헤더의 `**채널**` 필드에 채널 URL 하이퍼링크 추가.

```markdown
**채널**: [주토피아 - 어디에서도 들을 수 없는 술 이야기](https://www.youtube.com/@jutopia_TV)
```

**흐름**: `--channel URL` 인자 → `_save_manifest(channel_url=...)` → `_run_*_processing()` 에서 `ch_display = f"[{channel}]({ch_url})"` 조립 후 헤더에 주입.

**제한**: `--url` 단일 영상 모드는 channel_url을 API에서 받아오지 않으므로 링크 없이 채널명만 출력됨. 재처리 시 수동 패치 필요.
