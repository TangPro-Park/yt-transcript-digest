# 토큰 사용량 명세 — yt-digest

**기준 영상**: 약 15분 분량 유튜브 강연/인터뷰  
**분석 기준**: 실측 테스트 영상 `txa_8i-3cIs` (트랜스크립트 38,249 bytes / ~13,000자)  
**참고치 기준**: Claude Pro 일일 예산 추정 ~420,000T (Anthropic 미공개, Pro 기준 역산값)

---

## 1. 공통 입력(Input) 기준

모든 정리 모드는 동일한 입력 부하를 공유한다.

- **스크립트 원문**: 약 13,000T (3%)
- **프롬프트 템플릿**: 약 800T~1,300T (<1%)  (모드별 상이)
- **작업 지시 및 메타데이터**: 약 500T (<1%)
- **총 Input**: 약 **14,000T~14,500T (3%)**

토큰 예산 관리의 핵심은 Output 단계에 있다. Input은 어떤 모드를 선택하든 동일하게 발생한다.

---

## 2. 모드별 Output 및 총 토큰 비교

### Heavy 모드 (1차 정독용) — `--mode heavy`

- **구조**: 대주제·소주제별 심층 해설 + 스크립트 원문 100% 재출력 + 프리뷰·전체 요약·끝인사
- **예상 Output 글자 수**: 약 20,000자 (해설 + 원문 전체)
- **예상 Output 토큰**: 약 **20,000T (5%)**
- **총 예상 토큰**: 약 **34,500T (8%)**  → 하루 약 **12편** 처리 가능
- **핵심 특성**: 입력받은 13,000자 원문을 훼손 없이 전량 재출력하므로 Output 단계에서 토큰 폭발이 발생한다. 1차 정독·아카이빙 목적에 적합하다.

---

### Medium 모드 (2차 회독용) — `--mode medium`

- **구조**: 원문 출력 생략, 대주제·소주제별 심층 해설 + 전체 요약
- **예상 Output 글자 수**: 약 5,000자~7,000자 (순수 해설)
- **예상 Output 토큰**: 약 **6,000T~8,000T (1%~2%)**
- **총 예상 토큰**: 약 **21,000T (5%)**  → 하루 약 **20편** 처리 가능
- **핵심 특성**: Heavy 대비 Output 토큰 **약 60~65% 절감**. 원문 재출력 부담이 사라져 AI가 해설 품질에 리소스를 온전히 집중할 수 있는 가장 균형 잡힌 모드다.

---

### Compact 모드 (3차 압축용) — `--mode compact`

- **구조**: 동적 비례 기반 한눈 요약 + 핵심 주장 Top N (스토리텔링 포함) + 주요 키워드 심층 해설. 인사말·원문 일절 생략
- **예상 Output 글자 수**: 약 1,500자~2,000자 (고밀도 추출)
- **예상 Output 토큰**: 약 **1,500T~2,000T (<1%)**
- **총 예상 토큰**: 약 **16,000T (4%)**  → 하루 약 **26편** 처리 가능
- **핵심 특성**: Heavy 대비 Output 토큰 **90% 이상 절감**. 단, 분량이 적다고 표면 요약에 그쳐선 안 되며, 할당된 2,000자 안에 서술 깊이와 맥락을 고밀도로 압축하는 High-density generation이 필수다.

---

## 3. 모드별 요약 비교

- **Heavy**: Input ~14,500T (3%) + Output ~20,000T (5%) = 총 ~**34,500T (8%)**
- **Medium**: Input ~14,500T (3%) + Output ~7,000T (2%) = 총 ~**21,000T (5%)**
- **Compact**: Input ~14,500T (3%) + Output ~1,750T (<1%) = 총 ~**16,000T (4%)**

---

## 4. Claude Pro 일일 처리 가능 편수

- **Heavy**: 약 **12편** (편당 ~8%)
- **Medium**: 약 **20편** (편당 ~5%)
- **Compact**: 약 **26편** (편당 ~4%)

---

## 5. 모드 선택 가이드

- **원문 보존이 필수**이거나 나중에 원문을 다시 참조할 일이 있으면 → **Heavy**
- **심층 이해와 학습**이 목적이고 원문 재열람 계획이 없으면 → **Medium** (권장)
- **빠른 맥락 파악**이나 **토큰 예산이 촉박**할 때 → **Compact**
