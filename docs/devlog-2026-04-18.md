## 4. 핵심 설계 결정

### 4-1. 파이프라인 분리 구조

```
[Python] main.py → pending.json
                              ↓
[Claude] transcript 읽기 → 양식 적용 → .md 저장
```

Python은 데이터 수집만, LLM 처리는 Claude Code 세션이 직접 담당.

### 4-2. pending.json 스키마 (Python↔Claude 핸드오프)

```json
{
  "channel_name": "채널명",
  "channel_dir": "./output/채널명",
  "template": "./templates/deep_analysis.md",
  "pending": [
    {
      "video_id": "...",
      "title": "...",
      "published_at": "YYYY-MM-DD",
      "duration": "mm:ss",
      "url": "...",
      "channel_name": "...",
      "transcript_path": "./cache/transcripts/{id}.txt"
    }
  ],
  "skipped": ["video_id", ...]
}
```

### 4-3. processed 추적

`output/{채널명}/.processed.json` — video_id 리스트로 재처리 방지

## 5. CLI 인터페이스 (main.py)

```bash
# 단일 URL — API 키 불필요
python main.py --url "https://youtu.be/xxxx"

# 모드 1: 최신 미처리 영상 1개
python main.py --channel "URL" --latest

# 모드 2: 기간 내 미처리
python main.py --channel "URL" --start 2026-01-01 --end 2026-04-18

# 모드 3: 키워드 검색
python main.py --channel "URL" --keyword "클로드" [--start ...] [--end ...]

# 모드 4: 기간 내 전체 재처리
python main.py --channel "URL" --start 2026-01-01 --end 2026-04-18 --all
```

- `--channel` 생략 시 `config.yaml`의 `channel_url` 사용
- `--end` 생략 시 오늘 날짜 자동 적용
